JavaScript解析引擎

2018-04-04

什么是JavaScript解析引擎？

对于静态语言（如Java、C++、C）来说，程序员编写的源代码经过编译器编译为另一种代码，如机器码或者字节码，而像JavaScript这种动态语言，解释器会直接解析源代码并得出运行结果。但这种界定并非绝对，比如V8中，为了提高JavaScript性能，在运行前会将JavaScript编译为本地的机器码（native machine code），然后再去执行机器码，这样速度会快很多。

PS：JavaScript引擎本身也是程序，比如V8是用C/C++写的。

与ECMAScript的关系

JavaScript是一段程序，我们写的JavaScript代码也是一段程序，如何让程序读懂程序？这就需要一套规则，ECMAScript就是这套标准规则。
但由于市面上浏览器厂商很多，而且基于历史和竞争等原因，这些厂商在自己的JavaScript引擎的实现过程中加入一些自己的特色，没有完全按照标准实现，所以才有了浏览器兼容性问题。

与浏览器的关系

JavaScript引擎是浏览器组成部分。除此，浏览器还需要做很多事情：解析页面、渲染页面、Cookie管理、历史记录等等。
